# Postgres Bootstrap Assets

Generated by `bun run db:render:postgres`.

Contents:

- `platform-bootstrap.sql`: role, schema, RLS, and curated `api` projection bootstrap SQL
- `transaction-context.sql`: transaction-local request context statements for tenant/actor/request metadata
- `compose.yaml`: local Postgres bootstrap for development and security review

Recommended local flow:

1. `export PATH="$HOME/.bun/bin:$PATH"`
2. `bun run db:render:postgres`
3. `docker compose -f ops/postgres/compose.yaml up -d`
4. Apply domain migrations with the migrator role after reviewing the generated SQL.

Notes:

- The bootstrap SQL assumes canonical platform schemas such as `crm`, `sales`, `marketing`, and `booking`.
- Curated `api` views are intended for isolated plugins and sidecar services; same-process trusted plugins should still go through host services and repositories.
- Runtime roles remain non-owning and do not receive `BYPASSRLS`.
